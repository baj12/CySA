# AGENTS.md

R package extracted from CyDa: a Shiny dashboard (`clusterSelector`) for interactive selection and visualization of flow-cytometry clusters on `SingleCellExperiment` objects.

## Essential commands

```r
# Load and test
devtools::load_all()
devtools::test()
devtools::test_file("tests/testthat/test-<name>.R")

# Regenerate NAMESPACE / man pages from roxygen comments
devtools::document()

# Full package verification
devtools::check()

# Install / build
devtools::install()
devtools::build()

# README.md is generated from README.Rmd
devtools::build_readme()

# Documentation site
pkgdown::build_site()
```

## Key generated files

- `NAMESPACE` is generated by roxygen2 — do not edit by hand. Run `devtools::document()` after changing roxygen comments.
- `README.md` is generated from `README.Rmd` — edit `README.Rmd`, then run `devtools::build_readme()`.

## Architecture

- `R/clusterSelector.R` — main Shiny app factory (`clusterSelector()`), ~1900 lines.
- `R/shinyAppFunctions.R` — helper functions used by the app.
- `R/prepClusterSelector.R` — `prepClusterSelectorData()` for subsampling and pre-computation.
- `R/plotSOMScatter.R` and `R/plotScatterBJ.R` — plotting helpers copied from CyDa.
- `R/CySA-package.R` — central package imports and `CySA_default_cluster_cols()`.
- `inst/dev/` — working analysis notes (`claude.discussion.md`), not installed.

## App contract

- `clusterSelector()` returns a Shiny app object; launch with `shiny::runApp(app)`.
- It accepts an `env` argument and mutates `outputList` in that environment so callers can retrieve selections after the app closes. Preserve this unless explicitly replacing it.

## Dependencies and imports

- Add new dependencies with `usethis::use_package("pkg")`.
- Prefer `pkg::function()` calls or targeted `@importFrom` roxygen tags over broad `@import`. `devtools::check()` currently warns about namespace masking from whole-package imports.
- `ggthemr` is archived on CRAN but is currently imported. Theme calls should be made configurable or replaced.
- `raster` is not in `DESCRIPTION`; the app currently receives `somRasterObj` from the caller. Add `raster` via `usethis::use_package("raster")` only if the code starts constructing raster objects itself.

## Code style

- RStudio project enforces 2-space indentation, spaces for tabs, trailing-whitespace stripping, and POSIX line endings.
- Use roxygen2 markdown (`Roxygen: list(markdown = TRUE)` in `DESCRIPTION`).

## Hazards

- No `source()` calls or hardcoded absolute paths — all helpers must live in the package namespace.
- Several helpers originally referenced a global `sce`. Pass `sce` and `metaD` explicitly.
- The test suite (`tests/testthat/test-smoke.R`) is currently only a placeholder. Meaningful tests need real or mocked `SingleCellExperiment` inputs with SOM metadata.

## Secrets

- `.env` contains API keys. Do not commit it or include its contents in any file.
